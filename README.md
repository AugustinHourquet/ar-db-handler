# ar-db-handler

SQLite data-access layer for the annual report pipeline.

`ar-db-handler` owns the full database layer for two SQLite databases used by
the annual report pipeline. It is a **pure data-access module** — it knows
nothing about EDGAR, GCS, scraping logic, or evaluation logic. Every other
module in the pipeline that needs to read or write to these databases imports
from here.

---

## What it does (and does not)

**Does:**

- Defines and applies the full schema for `filings.db` and `metrics.db`.
- Provides `init_filings_db(path)` and `init_metrics_db(path)` that create
  tables if they do not exist, enable WAL mode, and enforce foreign-key
  constraints.
- Provides all read and write helpers (upserts, queries) for every table.
- Exposes a clean public API via `ar_db_handler.__init__`.

**Does not:**

- Know anything about EDGAR, GCS, scraping, or evaluation logic.
- Manage connections across threads — callers manage their own connections.
- Enforce cross-database foreign keys. `metrics.evaluations.filing_id`
  references `filings.filings.filing_id` only as a plain text field; there
  is no FK between the two databases.

---

## The two databases

### `filings.db`

Populated by scrapers, read by the evaluator. Tables:

- **`scraper_runs`** — one parent row per pipeline run (`worker_id IS NULL`)
  plus one row per worker (`worker_id` set, `country`/`config` NULL).
- **`companies`** — company reference data.
- **`filings`** — one row per `(company_id, fiscal_year)`, with a UUID
  `filing_id` generated by the scraper. The natural key
  `(company_id, fiscal_year)` is `UNIQUE`.
- **`filing_files`** — one row per concrete file produced for a filing
  (PDF, XBRL, …), with `scrape_status` of `PENDING | SCRAPED | FAILED`.
  The natural key is `(filing_id, file_type, form_type)`.

### `metrics.db`

Populated by the evaluator. Tables:

- **`evaluations`** — top-level metric snapshot for one evaluator run on
  one filing.
- **`evaluation_scores_by_statement`** — per-statement breakdown,
  linked to `evaluations` by FK.

## Public API

All callers import from the top-level package:

```python
from ar_db_handler import (
    # init
    init_filings_db, init_metrics_db,

    # filings.db writes
    upsert_run, upsert_worker, upsert_company,
    upsert_filing, upsert_filing_file,
    AlreadyScrapedError,

    # filings.db reads
    get_filing, get_filing_file,
    get_scraped_pairs, list_companies,

    # metrics.db writes
    write_evaluation, write_evaluation_scores_by_statement,

    # record dataclasses
    RunRecord, WorkerRecord, CompanyRecord,
    FilingRecord, FilingFileRecord,
    EvaluationRecord, StatementScoreRecord, ScrapedPair,
)
```

### Connections are caller-owned

`init_filings_db(path)` and `init_metrics_db(path)` return an open
`sqlite3.Connection`. The caller is responsible for closing it. Every write
helper takes a connection as its first argument and commits its own write —
if you need a larger transactional unit, wrap multiple calls in
`with conn:`.

## Usage

### Initialise

```python
from ar_db_handler import init_filings_db, init_metrics_db

filings = init_filings_db("data/filings.db")
metrics = init_metrics_db("data/metrics.db")
```

### Record a run + workers (scraper side)

```python
from ar_db_handler import RunRecord, WorkerRecord, upsert_run, upsert_worker

upsert_run(filings, RunRecord(
    run_id="run-2026-01-01",
    parent_run_id=None,
    country="US",
    started_at="2026-01-01T00:00:00",
    finished_at=None,
    status="RUNNING",
    config='{"max_workers": 3}',
    worker_count=3,
))

for w in (1, 2, 3):
    upsert_worker(filings, WorkerRecord(
        run_id=f"run-2026-01-01-w{w}",
        parent_run_id="run-2026-01-01",
        worker_id=w,
        started_at="2026-01-01T00:00:00",
        finished_at=None,
        status="RUNNING",
    ))
```

### Write a filing + its files

```python
from ar_db_handler import (
    CompanyRecord, FilingRecord, FilingFileRecord,
    upsert_company, upsert_filing, upsert_filing_file,
)
import uuid

upsert_company(filings, CompanyRecord(
    company_id="C001", name="Acme Corp",
    ticker="ACME", exchange="NASDAQ", country="US",
    updated_at="2026-01-01T00:00:00",
))

filing_id = str(uuid.uuid4())
upsert_filing(filings, FilingRecord(
    filing_id=filing_id,
    company_id="C001",
    fiscal_year=2024,
    filing_date="2025-02-15",
    reporting_date="2024-12-31",
    reporting_period="FY",
))

upsert_filing_file(filings, FilingFileRecord(
    file_id=str(uuid.uuid4()),
    filing_id=filing_id,
    run_id="run-2026-01-01-w1",
    worker_id=1,
    file_type="PDF",
    form_type="10-K",
    gcs_path="gs://my-bucket/C001/...pdf",
    url="https://www.sec.gov/...",
    scrape_status="SCRAPED",
    scraped_at="2026-01-01T00:10:00",
))
```

### `AlreadyScrapedError` and the expected caller pattern

`upsert_filing_file` raises `AlreadyScrapedError` when the
`(filing_id, file_type, form_type)` triple already has a row with
`scrape_status == 'SCRAPED'` and `force=False`. **The scraper is expected
to check `get_filing_file()` before attempting the download** — this
exception is a defence-in-depth signal, not a routine control-flow path.

```python
from ar_db_handler import get_filing_file, upsert_filing_file, AlreadyScrapedError

existing = get_filing_file(filings, filing_id, "PDF", "10-K")
if existing is not None and existing.scrape_status == "SCRAPED":
    # Skip the download; the file is already on GCS.
    return

# … download …

try:
    upsert_filing_file(filings, record)
except AlreadyScrapedError:
    # A concurrent worker scraped it first; bail out gracefully.
    return
```

To re-scrape (e.g. the GCS file was deleted, or you want to force a
refresh), pass `force=True`:

```python
upsert_filing_file(filings, record, force=True)
```

`PENDING` and `FAILED` rows are always replaced by a new `upsert_filing_file`
call regardless of `force`.

### `get_scraped_pairs()` — the evaluator's entry point

This is the primary discovery query. It returns every filing for which
**both** a PDF and an XBRL file have `scrape_status = 'SCRAPED'` — i.e.
every filing that is ready to evaluate.

```python
from ar_db_handler import get_scraped_pairs

pairs = get_scraped_pairs(filings)
for p in pairs:
    print(p.filing_id, p.company_id, p.fiscal_year)
    print("  PDF :", p.pdf_gcs_path)
    print("  XBRL:", p.xbrl_gcs_path)

# Optional filters:
pairs_2024 = get_scraped_pairs(filings, fiscal_year=2024)
pairs_acme = get_scraped_pairs(filings, company_id="C001")
pairs_one  = get_scraped_pairs(filings, company_id="C001", fiscal_year=2024)
```

### Write evaluation results

```python
from ar_db_handler import (
    EvaluationRecord, StatementScoreRecord,
    write_evaluation, write_evaluation_scores_by_statement,
)
import json, uuid

eval_id = str(uuid.uuid4())
write_evaluation(metrics, EvaluationRecord(
    evaluation_id=eval_id,
    filing_id=filing_id,
    company_id="C001",
    fiscal_year=2024,
    evaluated_at="2026-02-01T00:00:00",
    pdf_source="gs://...",
    xbrl_source="gs://...",
    scope=json.dumps(["IncomeStatement", "BalanceSheet", "CashFlow"]),
    xbrl_facts_scope=120, pdf_facts_scope=110,
    matched=100, missed=10, spurious=5,
    tier1_matches=80, tier2_matches=20,
    coverage=0.91, precision=0.95, recall=0.90, f1=0.92,
    exact_match_rate=0.85, within_1pct_rate=0.93, within_5pct_rate=0.97,
    output_json=json.dumps({...}),
    output_diff="...",
))

write_evaluation_scores_by_statement(metrics, [
    StatementScoreRecord(
        evaluation_id=eval_id,
        statement="IncomeStatement",
        coverage=0.9, precision=0.92, recall=0.91, f1=0.915,
        exact_match_rate=0.85, within_1pct_rate=0.93, within_5pct_rate=0.97,
    ),
    # …one per statement…
])
```

> Note: `scope`, `output_json`, and `output_diff` are stored as **already
> serialised strings**. The caller does the `json.dumps`.

`write_evaluation` is `INSERT` (not `INSERT OR REPLACE`); a duplicate
`evaluation_id` raises `sqlite3.IntegrityError`. Use a fresh UUID per
evaluator run, which keeps the metrics table append-only.

---

## Upsert rules at a glance

| Helper                 | Conflict key                                | Behaviour                                                 |
|------------------------|---------------------------------------------|-----------------------------------------------------------|
| `upsert_run`           | `run_id`                                    | `INSERT OR IGNORE` — original survives                    |
| `upsert_worker`        | `run_id`                                    | `INSERT OR IGNORE` — original survives                    |
| `upsert_company`       | `company_id`                                | `INSERT OR IGNORE`                                        |
| `upsert_filing`        | `(company_id, fiscal_year)`                 | `INSERT OR IGNORE` — filing event is immutable            |
| `upsert_filing_file`   | `(filing_id, file_type, form_type)`         | See logic below                                           |
| `write_evaluation`     | `evaluation_id`                             | `INSERT` only — duplicate raises `IntegrityError`         |

`upsert_filing_file` logic:

- No existing row → insert.
- Existing row, status != `SCRAPED` → `INSERT OR REPLACE`.
- Existing row, status == `SCRAPED`, `force=False` → raise
  `AlreadyScrapedError`.
- Existing row, status == `SCRAPED`, `force=True` → `INSERT OR REPLACE`.
